## The defect to file

- **Where:** staging, app launch, before the intro → login transition.
- **What:** `POST /device/register` returns HTTP 404; the app reacts with a modal
  `THÔNG BÁO — Đã có lỗi xảy ra, Quý khách vui lòng thử lại sau! (58)` over the intro
  screen. Login is unreachable, so every login test is blocked.
- **Not reproducible by hand right now:** the same flow driven manually with
  `agent-device` (including after a data wipe) reaches the home tab bar, which further
  isolates the problem to the register call, not the UI.
